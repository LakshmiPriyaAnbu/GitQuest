⏩ Fast-forward merge
When main hasn't moved since you branched, Git just slides the pointer forward. History stays a single straight line — no merge commit.
🔀 Three-way merge
When both branches gained commits, Git weaves them with a merge commit that has two parents — tying the two paths back together.
⚔️ Merge conflict
When both branches edit the same lines, Git can't decide. It's a boss battle — you choose which version wins, then commit the resolution.
